title Trade Engine Sequence Diagram

participant Message Queue
participant Trade Engine
participant Cache Service

Message Queue-> Trade Engine:Pull processing order task
Trade Engine-> Cache Service:Distributed lock queue
Cache Service--> Trade Engine:
opt Lock successful
    alt Task is LIMIT order
        Trade Engine->Cache Service:Get target queue at task price
        Cache Service-->Trade Engine:
        Trade Engine->Trade Engine:Get matched list (exact price only, using takeWhile)
    else Task is MARKET order
        Trade Engine->Cache Service:Get entire target book (sorted by price)
        Cache Service-->Trade Engine:
        Trade Engine->Trade Engine:Get matched list (across price levels, using takeWhile)
    end

    opt Matched list is not empty
        loop For each targetOrder in matched list
            Trade Engine->Trade Engine:Calculate matchQty (Math.min)
            Trade Engine->Trade Engine:Deduct quantities
            Trade Engine->Trade Engine:Create OrderDetailPojos (Maker price & Taker/Maker IDs)
            Trade Engine->Cache Service:Update/Remove targetOrder from cache (if filled)
            Cache Service-->Trade Engine:
            opt targetOrder is filled
                Trade Engine-->Message Queue:Insert targetOrder into filled queue
            end
        end
        Trade Engine->Trade Engine:Update task order quantity and status
        Trade Engine->Cache Service:Update/Re-queue task order in cache (if LIMIT and processing)
        Cache Service-->Trade Engine:
        opt task order is filled
            Trade Engine-->Message Queue:Insert task order into filled queue
        end
    end
    Trade Engine->Cache Service:Unlock queue
    Cache Service-->Trade Engine:
end
Trade Engine-->Trade Engine:Return Task
opt Task status in processing
    Trade Engine-->Message Queue:Push processing order task
end
